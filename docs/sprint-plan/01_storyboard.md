# Storyboard — سناریوهای کاربری

> این فایل «داستان‌های قابل اجرا» می‌سازد تا مسیر تبدیل Spec به Backlog شفاف باشد.
> هر سناریو باید خروجی مشخص: **کدام Epic/Story** و **چه معیار موفقیت** داشته باشد.

## SB-01 — اولین تجربه: از صفر تا اولین چت با Agent

**پرسونا:** Founder / Admin  
**هدف:** در کمتر از ۱۵ دقیقه یک Agent بسازد و اولین مکالمه را انجام دهد.

**فریم‌ها**

1. ورود به داشبورد → ساخت Organization/Workspace
2. تنظیم API Keyها (AI Gateway) → تست اتصال مدل
3. ساخت Agent (نام، نقش، مدل پیش‌فرض)
4. ارسال پیام → دریافت پاسخ (با استریم)
5. مشاهده تاریخچه گفتگو و وضعیت اجرا

**خروجی‌های مورد انتظار (MVP)**

- Agent قابل ساخت/حذف/فعال-غیرفعال شدن
- Endpoint چت و استریم (SSE/WebSocket طبق Spec)
- Dashboard حداقلی برای همین مسیر

**معیار موفقیت**

- Time-to-first-chat < 15 دقیقه (محیط dev)
- P95 latency پاسخ متنی زیر 2s (برای پیام کوتاه، در dev)

**تبدیل به بک‌لاگ**

- Epic: E-001 (Core Engine MVP)
- Stories: US-001..US-004
- Tasks: هم‌راستا با Sprint 0 (T-001..T-008 در `docs/spect/09_TASKS/01-current-sprint.md`)

## SB-02 — حافظه پایه: Agent «یادش می‌ماند»

**پرسونا:** Admin / Power User  
**هدف:** Agent بتواند مکالمه را ذخیره کند و خلاصه‌سازی/بازیابی انجام دهد.

**فریم‌ها**

1. مکالمه چندمرحله‌ای → ذخیره short memory (Redis با TTL)
2. ذخیره conversation memory (PostgreSQL)
3. خلاصه‌سازی خودکار پس از N پیام
4. نمایش تاریخچه/خلاصه در UI

**معیار موفقیت**

- ذخیره/بازیابی تاریخچه بدون از دست رفتن پیام‌ها
- زمان بازیابی history در P95 < 200ms (dev target)

**تبدیل به بک‌لاگ**

- Epic: E-010 (Memory L1/L2)
- Stories: US-010..US-013

## SB-03 — مسیر «Skill»: اجرای یک ابزار با Permission و لاگ

**پرسونا:** Admin / Developer  
**هدف:** یک Skill نصب/فعال شود، مجوز بگیرد، اجرا شود، و قابل ردیابی باشد.

**فریم‌ها**

1. انتخاب Skill از Registry (داخلی)
2. نمایش Permissionها و تایید کاربر (HITL ساده)
3. اجرا روی یک ورودی (مثلاً summarize / fetch)
4. ثبت نتیجه + لاگ اجرا + خطاها

**معیار موفقیت**

- هر اجرا traceable (request id + log)
- Permission gate فعال (بدون تایید اجرا نشود)

**تبدیل به بک‌لاگ**

- Epic: E-020 (Skill Engine MVP)
- Stories: US-020..US-024

## SB-04 — اتصال کانکتور: ingest دانش به حافظه

**پرسونا:** Admin  
**هدف:** یک منبع (مثل Google Drive) وصل شود و داده وارد Memory شود.

**فریم‌ها**

1. اضافه کردن Connector و احراز هویت
2. انتخاب scope (پروژه/شرکت/شخصی)
3. شروع ingest → مشاهده progress
4. جستجو و بازیابی دانش در چت

**معیار موفقیت**

- ingest قابل مشاهده و قابل retry
- نتایج قابل جستجو با latency معقول

**تبدیل به بک‌لاگ**

- Epic: E-043 (Integration Hub / Connectors)

## SB-05 — Organization و Workspace: تیم چندنفره

**پرسونا:** Org Admin  
**هدف:** ساخت tenant/Workspace، دعوت کاربران، و اعمال نقش‌ها.

**فریم‌ها**

1. ساخت Organization → ساخت Workspace
2. ارسال دعوت‌نامه → پذیرش → ورود
3. assign role (Admin/Member/Viewer)
4. محدود شدن دسترسی‌ها طبق RBAC

**معیار موفقیت**

- کاربر جدید در < 5 دقیقه onboard شود (dev)
- RBAC پیش‌فرض درست enforce شود

**تبدیل به بک‌لاگ**

- Epic: E-030, E-031, E-006

## SB-06 — Governance: Approval و Budget

**پرسونا:** Security Admin / Auditor  
**هدف:** اکشن‌های حساس (مثل نصب پلاگین، تغییر policy، ingest گسترده) نیاز به تایید داشته باشند و هزینه کنترل شود.

**فریم‌ها**

1. کاربر یک اکشن حساس درخواست می‌دهد
2. سیستم request را در approval queue قرار می‌دهد
3. Admin تایید/رد می‌کند
4. اجرای اکشن + ثبت audit log + مصرف budget

**معیار موفقیت**

- بدون approval، اکشن اجرا نشود
- audit log قابل جستجو باشد

**تبدیل به بک‌لاگ**

- Epic: E-032, E-050

## SB-07 — Marketplace: نصب پلاگین و (بعداً) پرداخت

**پرسونا:** User / Publisher  
**هدف:** نصب/آپدیت پلاگین، و در نسخه پیشرفته خرید/پرداخت.

**فریم‌ها**

1. مرور marketplace → انتخاب پلاگین
2. نمایش مجوزها و سیاست‌ها → تایید
3. نصب/فعال‌سازی → اجرا در چت
4. (پیشرفته) خرید پلاگین پولی + فاکتور/پرداخت

**معیار موفقیت**

- نصب پلاگین بدون شکستن سیستم
- درآمد/سهم ناشر قابل محاسبه باشد

**تبدیل به بک‌لاگ**

- Epic: E-024, E-040, E-023

## SB-09 — Enterprise: SSO و محیط Private

**پرسونا:** Enterprise Admin  
**هدف:** SSO فعال شود و deployment در محیط private/VPC قابل ارائه باشد.

**فریم‌ها**

1. تنظیم SSO (OIDC/SAML) → تست login
2. enforce policy برای دسترسی‌ها
3. deploy روی Kubernetes در شبکه خصوصی
4. مشاهده SLA/Monitoring و runbook

**معیار موفقیت**

- ورود با SSO بدون کاربر محلی
- audit و لاگ‌ها قابل ارائه برای compliance

**تبدیل به بک‌لاگ**

- Epic: E-050, E-051, E-052
