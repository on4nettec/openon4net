# Competitive Positioning — Open on4net (O₂N)

> **فایل:** 00_VISION/07-competitive-positioning.md  
> **نسخه:** 1.0 | **تاریخ:** July 2026

---

## ۱) پیام اصلی محصول

O₂N یک **Enterprise AI Platform / AI Operating System** است که به کسب‌وکارها اجازه می‌دهد **کارکنان هوش مصنوعی** را مثل کارمند واقعی:
- ایجاد کنند
- آموزش دهند
- مدیریت و کنترل کنند (governance)
- ارزیابی کنند (KPI/Outcome)
- و در سطح سازمان، همکاری و یادگیری را نهادینه کنند

این یک Chatbot نیست.  
این صرفاً یک Agent runner هم نیست.  
این یک «سیستم اداره‌کردن شرکت با تیم Digital Employeeها» است.

---

## ۲) تفاوت با OpenClaw

### OpenClaw می‌گوید:
«من Agent اجرا می‌کنم.»

### O₂N می‌گوید:
«ما شرکت را اداره می‌کنیم.»

**تفاوت کلیدی:**
- OpenClaw معمولاً حول اجرای Agentها/Taskها می‌چرخد.
- O₂N حول **سازمان** می‌چرخد: Organization/Workspace، نقش‌ها، بودجه، approval chain، audit، KPI، گزارش‌دهی، و اثرگذاری بر خروجی‌های واقعی کسب‌وکار.

**خروجی مورد انتظار O₂N** فقط «پاسخ خوب» نیست؛
خروجی **عملیات سازمانی قابل ردیابی** است:
- چه کاری انجام شد؟
- چرا انجام شد؟
- چقدر هزینه داشت؟
- چه KPI را تغییر داد؟
- آیا نیاز به تأیید انسانی داشت؟
- چگونه قابل rollback است؟

---

## ۳) تفاوت با Hermes

### Hermes می‌گوید:
«من یاد می‌گیرم.»

### O₂N می‌گوید:
«کل سازمان یاد می‌گیرد.»

**تفاوت کلیدی:**
یادگیری و حافظه فقط برای یک Agent نیست؛ برای کل شرکت است:
- دانش شرکت (Company Knowledge)
- حافظه پروژه‌ها (Project Memory)
- گراف روابط سازمانی (Memory Graph)
- نتیجه‌محوری (Outcome Engine / KPI)

هدف این است که تجربه‌های سازمانی (تصمیم‌ها، اجراها، نتایج، خطاها) به دانش تبدیل شوند و برای همه Digital Employeeها قابل استفاده باشند؛ با مرزبندی دسترسی و حریم خصوصی.

---

## ۴) هم‌زمان «خوبی‌های Hermes و OpenClaw»

O₂N باید این دو ویژگی را هم‌زمان داشته باشد:

### ۴.۱) اجرای قابل اتکا (Runner/Execution Quality)
- اجرای ابزار/اکشن/ورک‌فلو با زمان‌بندی، retry، timeout
- observability و tracing کامل
- کنترل هزینه، rate limit، و fallback

### ۴.۲) یادگیری و بهبود سازمانی (Organizational Learning)
- Skill Engine برای تبدیل تکرارها به مهارت
- Memory Engine چندلایه + گراف
- Outcome Engine برای سنجش اثر واقعی
- Governance برای جلوگیری از اشتباهات پرهزینه

---

## ۵) زنجیره ارتباطی (Communication Chain)

در O₂N ارتباط فقط «کاربر ↔️ یک Agent» نیست. ما یک زنجیره ارتباطی سازمانی داریم:
- user → CEO AI → department lead (مثلاً Marketing AI) → sub-agents (Designer/Copy/Publisher)
- agent → agent (sync/async) با governance check
- agent → human approval queue (در تصمیم‌های حساس)
- system → dashboards/reports (daily/weekly)

این زنجیره باید:
- قابل ردیابی (trace_id) باشد
- قابل حسابرسی (audit) باشد
- قابل بازگشت (rollback) باشد
- و قابل اندازه‌گیری (KPI/Outcome) باشد

---

## ۶) اکوسیستم ابزار/پلاگین و اقتصاد (Marketplace)

O₂N باید طوری طراحی شود که:
- هر توسعه‌دهنده بتواند **tool/plugin** بسازد (SDK + تست + review)
- مصرف‌کننده با یک **Credit/Coin داخلی** هزینه مصرف را پرداخت کند
- سازنده از مصرف منتفع شود (revenue share)

> نکته مهم: این «Coin» در نسخه v1 به‌عنوان **اعتبار داخلی (credits)** تعریف می‌شود (نه توکن قابل معامله عمومی) تا ریسک‌های حقوقی/سازگاری کاهش یابد. امکان اتصال به سیستم‌های پرداخت/کیف پول در آینده به‌عنوان گزینه‌ی جداگانه دیده می‌شود.

---

## ۷) تعریف Done برای “پلتفرم اداره‌کردن شرکت”

پلتفرم زمانی واقعاً به هدف نزدیک است که بتواند:
- یک سازمان را ایجاد کند و تیمش را onboard کند
- چند Digital Employee با نقش واقعی بسازد
- یک دستور «business-level» بگیرد (مثلاً کمپین + کنترل بودجه + تحلیل نتیجه)
- آن را به کارهای قابل اجرا تقسیم کند
- اجرا کند و نتیجه را گزارش دهد
- هزینه/اجازه/لاگ را enforce کند
- و دانش حاصل را در حافظه سازمانی ذخیره کند

---

> **خلاصه:** O₂N نه “Agent runner”، نه “Chatbot”، نه “یک Agent هوشمند”؛ بلکه یک پلتفرم سازمان‌محور است که اجرای قابل اتکا + یادگیری سازمانی + اکوسیستم توسعه‌دهندگان + اقتصاد داخلی را یکجا ارائه می‌دهد.
