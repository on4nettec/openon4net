# Onboarding Guide — Open on4net

> **فایل:** 09_TASKS/06-onboarding.md
> **نسخه:** 1.0

---

## ۱. جریان کلی onboarding یک شرکت جدید

```
ثبت‌نام → ایجاد Organization → ساخت اولین Agent → آموزش Agent → شروع کار
```

---

## ۲. مرحله ۱: ثبت‌نام (۵ دقیقه)

```yaml
steps:
  1. ورود با Email یا SSO (Google, Microsoft)
  2. تأیید ایمیل
  3. اطلاعات شرکت: نام، صنعت، تعداد کارمندان
  4. انتخاب پلن: Starter (رایگان) یا Team
  5. ✅ Organization ساخته شد
```

### صفحه ثبت‌نام:
```
┌──────────────────────────────────┐
│  به O₂N خوش آمدید                │
│                                  │
│  ایمیل: soroush@on4net.com      │
│  شرکت: on4net                   │
│  صنعت: تکنولوژی                  │
│  تعداد کارمندان: ۵              │
│                                  │
│  [ادامه با Google] [ثبت‌نام]    │
└──────────────────────────────────┘
```

## ۳. مرحله ۲: Workspace Setup (۱۰ دقیقه)

### بعد از ثبت‌نام:

```yaml
steps:
  1. نام Workspace: "on4net workspace"
  2. انتخاب دپارتمان‌های اولیه:
     □ Marketing (بازاریابی)
     □ Sales (فروش)
     □ Support (پشتیبانی)
     □ HR (منابع انسانی)
     □ Finance (مالی)
     □ Legal (حقوقی)
     ☑ Programmer (برنامه‌نویسی)
     □ SEO (سئو)
  3. تنظیم API Keys برای مدل‌ها (یا استفاده از keys پیش‌فرض O₂N)
  4. دعوت تیم (یا رد کردن)
  5. ✅ Workspace آماده است
```

### صفحه دعوت تیم:
```
┌──────────────────────────────────┐
│  تیم خود را دعوت کنید            │
│                                  │
│  ایمیل همکاران:                  │

│  [                         ] [+]│
```

## ۴. مرحله ۳: اولین Agent (۱۵ دقیقه)

### O₂N به شما پیشنهاد می‌دهد:

```yaml
steps:
  1. "بر اساس صنعت شما (تکنولوژی)، این Agentها پیشنهاد می‌شوند:"
     
     پیشنهاد شده:
     □ 🤖 CEO AI — مدیر عامل دیجیتال
     ☑ 👨‍💻 Programmer AI — برنامه‌نویس ارشد
     □ 📧 Marketing AI — مدیر بازاریابی
     □ 💰 Sales AI — مدیر فروش
  
  2. انتخاب Programmer Agent
  3. تنظیمات:
     - نام: "Pouya (Programmer AI)"
     - مدل اصلی: Claude 3.5 Sonnet
     - بودجه ماهانه: $۱۰۰
     - دسترسی‌ها: GitHub, Docker
  4. ✅ اولین Agent ساخته شد
```

## ۵. مرحله ۴: آموزش Agent (۲۰ دقیقه)

### Agent نیاز به context دارد:

```yaml
steps_auto:
  - Agent می‌پرسد: "شرکت شما چه محصولی تولید می‌کند؟"
    پاسخ: "یک AI Operating System به اسم O₂N"
    
  - Agent می‌پرسد: "تکنولوژی‌های اصلی شما چیست؟"
    پاسخ: "Node.js, TypeScript, PostgreSQL, Next.js"
    
  - Agent می‌پرسد: "مخزن کد شما کجاست؟"
    پاسخ: "github.com/on4nettec/openon4net"
    
  - Agent می‌پرسد: "آیا مستندات فنی دارید؟"
    پاسخ: "بله، در docs/spect/"
```

### بعد از آموزش:
```
Agent: "عالی! من الآن می‌دانم:
- شما روی O₂N کار می‌کنید
- تکنولوژی‌های اصلی: Node.js, TypeScript
- ریپازیتوری: github.com/on4nettec/openon4net
- مستندات در docs/spect/

حاضرم شروع کنم. اولین task چیست؟"
```

## ۶. مرحله ۵: اولین Task (۱۰ دقیقه)

```
شما: "یک code review برای Pull Request #۴۲ انجام بده"

Agent: "در حال بررسی PR #۴۲...
📝 تحلیل:
- ۳ فایل تغییر کرده (۱۲ خط اضافه، ۵ خط حذف)
- ساختار clean است
- یک نکته: تابع `getMemory` می‌تواند async باشد
- یک warning: متغیر unused در خط ۲۳

✅ پیشنهاد: Approve با یک نظر اصلاحی

آیا می‌خواهی این کار را به Skill تبدیل کنم؟
"code-review-pr" — بعد از این در ۲ ثانیه انجام می‌شود.
```

## ۸. Use Case Templates

### برای شروع سریع بر اساس نقش:

| نقش | اولین Agent | اولین Task |
|-----|-------------|------------|
| **توسعه‌دهنده** | Programmer AI | Code review یک PR |
| **مدیر محصول** | CEO AI | تحلیل وضعیت پروژه |
| **بازاریاب** | Marketing AI | طراحی کمپین |
| **پشتیبان** | Support AI | پاسخ به تیکت نمونه |
| **مدیر فروش** | Sales AI | بررسی pipeline فروش |
| **مدیر منابع انسانی** | HR AI | بررسی رزومه نمونه |

---

> **خلاصه:** Onboarding O₂N یک جریان ۵ مرحله‌ای در ~۴۵ دقیقه است: ثبت‌نام → Workspace → اولین Agent → آموزش → اولین Task. بعد از آن کاربر آماده کار است. Use Case Templates برای شروع سریع بر اساس نقش کاربر طراحی شده.
